El árbol genealógico de los lenguajes de marcas

SGML

El lenguaje SGML (Standard Generalized Markup Languages), deriva de GML, propuesto por IBM, un lenguaje que permitió a los usuarios trabajar en estilos de formato estandarizados para documentos electrónicos. SGML surgió para resolver problemas de interoperabilidad y gestión de información estructurada en distintos sistemas y plataformas, además de especificar las reglas para los elementos de marcado.
El lenguaje SGML es una forma más sencilla para escribir documentos usando etiquetas que ayudan a organizarlos. Gracias a esto, los archivos se pueden leer fácilmente tanto por personas como por ordenadores.



HTML

Tim Bernes Lee utilió SGML para crear un nuevo lenguaje de etiquetas que llamó Hypertext Makrkup Language, conocido como HTML. Este lenguaje fue desarrollado para estructurar y organizar el texto de manera legible mediante el uso de las etiquetas. HTML ofrece gran adaptabilidad,porque permite que el contenido web se visualice correctamente desde cualquier dispositivo.
HTML se usa comúnmente combinando CSS y JavaScript. Además, su uso principal puede ser para desarrollo de páginas web, para navegar por Internet, para diseños responsivos, etc.



XML

El lenguaje XML (eXtensible Markup Language), que permite almacenar, transmitir y reconstruir datos arbitrarios entre ordenadores (importar y exportar archivos), de manera sencilla y generalizada pero con una sintaxis más estricta; Fue creado por un comité del W3C o World Wide Web Consortium en 1996. Y surgió por la necesidad de un formato de lenguaje de marcado que fuera más flexible y personalizable que el SGML.
El lenguaje XML fue publicado oficialmente como una recomendación del W3C el 10 de febrero de 1998 y desde entonces es utilizado en diversas industrias y aplicaciones.



JSON

JSON o JavaScript Object Notation creado por Douglas Crockford en los principios de la década de los 2000, procedente de JavaScript y actualmente compite con XML, porque también se especializa en importar y exportar archivos; Aunque no se considere un lenguaje de marcas, ya que guarda el texto como datos o metadatos, y no utiliza ninguna etiqueta.



MarkDown

Markdown fue creado por John Gruber y Aaron Swartz en 2004. Originalmente buscaba ofecer una forma de convertir documentos escritos en texto plano para convertirlos fácilmente a HTML. En 2006 apareció una especificación más formal llamada “Markdown Extra”, agregando características adicionales, como tablas y definiciones de términos. Nació no como un sustituto de HTML, sino como una versión más simplificada de este, por lo que se usa para crear documentos que se pueden transformar en otros formatos, como HTML.



RFT

El Rich Text Format es un formato de archivo informático desarrollado por Microsoft en 1987 para el intercambio de documentos multiplataforma. La mayoría de los procesadores de texto pueden leer y escribir documentos RTF. No debe confundirse con el enriched text o su predecesor Rich Text, que son especificaciones completamente diferentes.
Richard Brodie, Charles Simonyi, y David Luebbert, miembros del equipo de desarrollo de Microsoft Word, desarrollaron el RTF original a mediados y finales de los 80. El primer lector y escritor de RTF enviado en 1987 como parte de Microsoft Word 3.0 para Macintosh, que implementó la especificación RTF versión 1.0. Todas las versiones posteriores de Microsoft Word para Macintosh, así como todas las versiones de Windows, pueden leer y escribir en formato RTF.
